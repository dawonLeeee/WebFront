문자열->숫자

        
        요소의 내용 또는 값을 얻어오면 기본적으로 문자열(String)으로 인식됨
        -> 덧셈 연산 시 더하기가 아닌 이어쓰기가 되는 문제 발생
    
        parseInt(문자열) : "정수" -> 정수("123"->123)
                            "실수" -> 정수("3.14" -> 3) --> 소수점 버림
    
        parseFloat(문자열) : "실수" -> 실수("3.14" -> 3)
                            "정수" -> 정수("3" -> 3)

        Number(문자열) : "실수" -> 실수("3.14" -> 3)
                            "정수" -> 정수("3" -> 3)

동등 비교 / 동일 비교 연산자

        동등 비교 연산자( == , !=)
            -> 값이 같으면 true(자료형 관계 없음)

            ex) true == '1' == 1

        동일 비교 연산자 ( ===, !==)
            -> 값과 자료형이 모두 같으면 true

문자열 내장 함수

        string.indexOf("찾고싶은문자열")
            -> string 내에서 "찾고싶은문자열"의 시작 인덱스를 반환.
                없으면 -1반환
                
        string.lastIndexOf("찾고싶은문자열")
            -> 뒤에서부터 검색하여 string 내에서 "찾고싶은문자열"의 시작 인덱스를 반환.
                없으면 -1반환


        string.substring(시작 인덱스)
            -> 문자열에서 시작 인덱스부터 끝까지 잘라서 반환
            ex) "Hello World".substring(2) --> "llo World"

        string.substring(시작 인덱스, 마지막 인덱스(미포함))
            -> 문자열 시작 인덱스부터 마지막 인덱스(미포함) 사이를 잘라서 반환


        string.split("구분자") : 문자열을 "구분자"를 기준으로 나누어 배열로 반환
            -> const str = "김밥, 샌드위치, 라면, 파스타, 햄버거, 스테이크";
                const arr = str.split(", ")
                 arr-->(6) ['김밥', '샌드위치', '라면', '파스타', '햄버거', '스테이크']

            * 문자열.split("") -> 문자열이 한 글자 모두 나누어져 배열로 반환

숫자 관련 내장 객체 / 내장 함수

        숫자(number) 타입 리터럴 표기법
        - 123(정수), 9.321(실수), Infinity(무한), NaN(Not a Number)

        숫자 관련 내장 객체 : Math
            - Math.random() : 0이상 1미만의 난수 발생

            (소숫점 "첫째" 자리에 적용-> 정수부분만 표현)
            - Math.ceil(값) : 올림
            - Math.floor(값) : 내림
            - Math.trunc(값) : 버림
            - Math.round(값) : 반올림

            - 숫자.toFixed(자릿수) : 숫자를 지정된 자릿수까지 반올림해서 표현
                ex) 3.141592.toFixed(2) == 3.14

isNan(값)

        값이 숫자가 아니면 true
        값이 숫자면 false
